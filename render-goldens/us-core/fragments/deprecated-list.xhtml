Set of codes that are needed for implementation of the US-Core CarePlan Profile. These codes are used as extensions to the FHIR ValueSet.
US Core Gender Identity Extension (StructureDefinition) retired

This extension is no longer a USCDI requirement and is deprecated. It SHOULD NOT be used for new or revised content. It is retained for historical/backward compatibility purposes. Implementers can use the HL7 standard extension instead

This extension represents an individual's sense of being a man, woman, boy, girl, nonbinary, or something else, ascertained by asking them what that identity is.
US Core Sex Extension (StructureDefinition) retired

This extension is no longer a USCDI requirement and is deprecated. It SHOULD NOT be used for new or revised content. It is retained for historical/backward compatibility purposes. Implementers SHOULD use the US Core Individual Sex Extension instead

The Sex Extension is used to reflect the documentation of a person's sex. Systems choosing to record sources of information should use the Provenance resource .

USCDI includes a data element for sex, intended to support the exchange of a sex value that is not characterized as sex assigned at birth or birth sex. This Sex extension supports USCDI. Sex assigned at birth or birth sex can be recorded using the more specific US Core Birth Sex Extension . Future versions of this extension may be informed by the content of the HL7 Cross Paradigm IG: Gender Harmony - Sex and Gender Representation, which may include additional guidance on its relationship to administrative gender ( Patient.gender ).
