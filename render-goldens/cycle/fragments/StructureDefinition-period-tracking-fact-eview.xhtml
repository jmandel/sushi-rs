Data
Period-tracking fact type
Cardinality: 1..1
CodeableConcept
 Identifies the kind of fact represented by this Observation, such as menstrual bleeding, menstrual flow, symptom, numeric pain severity, or basal body temperature.

Comment: Concrete profiles fix this code. Use this base profile only when this guide defines no more specific fact profile.

 value is bound to LOINC Codes (>1000 codes)
Date or timestamp for the source fact
Cardinality: 1..1
dateTime
 The date or time associated with the source fact. Producers MAY use date precision (for example, 2026-06-24) when the source stores a calendar-day fact, or full timestamp precision when the source stores a specific time. Producers SHALL NOT invent a time solely to satisfy this profile.
Recorded fact value
Cardinality: 1..1
Choice
 The value recorded by the source application for this fact. Concrete profiles narrow this to the appropriate FHIR value type.

Comment: Use exactly one value for each fact. If the source did not record a value, omit the fact instead of sending an empty or default value.

 value is a choice of:
  • Quantity
  • CodeableConcept
  • string
  • boolean
Protocol
Only final facts are exported
Cardinality: 1..1
code
 Period Tracking exports include finalized facts. Producers omit draft, preliminary, entered-in-error, and amended records from the shared Bundle.

Comment: If the source record is not ready to share as the user's current recorded fact, do not include it in the Bundle.

 value is fixed to final from ObservationStatus (7 codes)
Optional source application or device
Cardinality: 0..1
Reference
 Optional reference to a Device resource that identifies the source application, wearable, thermometer, or other system that produced or measured this fact.

Comment: Use this when it helps receivers identify the source of the fact without adding unnecessary identifying information.

 value must point to a Device
Context
Optional person reference
Cardinality: 0..1
Reference
 Optional reference to the Patient resource for the person whose period-tracking data is represented. The Bundle is still scoped to one person when Patient resources and subject references are omitted.

 value must point to a Patient
